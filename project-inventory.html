Inventory Management Dashboard

My Role
Exploratory Data Analysis · Forecast Model Development · Dashboard Design
Tech Used
Python · SQL · Scikit-learn · FastAPI · Pandas · Matplotlib

Overview

A demand-forecasting and inventory-planning system built for a large retail enterprise managing hundreds of SKUs. The tool improves visibility into sales patterns, forecasts inventory needs, and surfaces stockout risks early. The dashboard equips planners with clear, actionable insights rather than scattered spreadsheets.

Key objectives included:

  • Predicting baseline and seasonal sales more accurately.
  • Flagging potential stockouts ahead of time.
  • Recommending optimal reorder timing.
  • Presenting SKU-level insights through an interactive, intuitive interface.

Key Features

Exploratory Analysis

  • Identified seasonal peaks and demand cycles across product families.
  • Analyzed correlation structures to group highly similar SKUs.
  • Designed exception logic for new launches, replacements, and zero-sales periods.

Predictive Modeling

  • Built time-series models with custom lookback windows for baseline forecasts.
  • Applied multiplicative seasonality indices to capture monthly demand variation.
  • Segmented SKUs into continuous, inactive, and sparse groups, each with tailored forecasting logic to avoid overfitting.

Dashboard & API Layer

  • Inventory & Alerts: SKU table with stock levels, metadata, and rule-based stockout alerts.
  • Forecasting & Planning: Historical vs. predicted sales with component/size-level breakdowns and reorder timing recommendations.
  • APIs & Integration: FastAPI endpoints for forecast and SKU metadata, returning JSON formatted for clean UI rendering.
