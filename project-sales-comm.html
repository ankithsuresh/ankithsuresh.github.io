Sales Communication Analytics

My Role
LLM Architecture Development · Prompt Design · Cost/latency Optimization
Tech Used
Python · FastAPI · MongoDB Atlas · OpenAI GPT-4o

Overview

Sales context often gets buried in long email threads, making it hard to track account status, next steps, and ownership. This communication-analytics app ingests consented email data and produces structured summaries at the Account, Employee, Contact, and Single-Email levels.

The dashboard surfaces clear sections (Overview, Key Contacts & Takeaways, Action Items, Contact Overview, Alerts) plus activity KPIs (reply rate, average response time, message volume) so managers can quickly spot risks and next steps.

Key Features

Email Data Ingestion

  • Unified raw emails into consistent entities (accounts, contacts, employees, threads).
  • Standardized fields to enable reliable retrieval, joins, and roll-ups across reports.

Report-ready packets for summarization

  • Prepared report-specific JSON that included metadata and activity signals (e.g., reply rates, recency).
  • Kept outputs predictable so the frontend rendered cleanly with minimal post-processing.

LLM chain for reliability

  • Built a modular LLM pipeline that generated concise, sectioned summaries per report type.
  • Applied light structural validation to keep outputs aligned with the expected schema.

Token & cost controls

  • Implemented token counters and auto-adjust mechanisms to keep prompts within context limits.
  • Provided user-visible messaging when truncation or batching was applied.

APIs & dashboard

  • Exposed FastAPI endpoints to power report generation and KPI retrieval.
  • Had the frontend consume validated JSON to render summaries and activity KPIs.
