Data Analytics Assistant

My Role
LLM Architecture Development · Prompt Design · Cost/latency Optimization
Tech Used
Python · FastAPI · Elasticsearch · AWS · OpenAI

Overview

This conversational data analytics assistant app lets users query and analyze a large corpus of U.S. company data (financials + firmographics) in natural language. The assistant understands both company-specific lookups and exploratory, filter-driven lists, then returns structured answers with tables/charts and short, human-readable insights. The system emphasizes clarity, speed, and low cost while keeping responses consistent enough for a production UI.

Key Features

Two query modes

  • Specific Queries: Direct lookups for a known company or set of companies (e.g., revenue history, industry, HQ).
  • General Queries: Exploratory lists/aggregations across filters such as industry, location, founding year, revenue, EBITDA, and growth.

Natural-language understanding with disambiguation

  • Cleans and clarifies user questions to remove ambiguity.
  • Resolves entities (company names/IDs), dates, locations, and industries.
  • Normalizes industries to standard taxonomies (e.g., NAICS) for consistent results.

Intent detection & query planning

  • Classifies the user’s request (overview, financials, lists/aggregates, etc.) and maps it to relevant field sets.
  • Produces a retrieval plan that safely queries the search index and prepares the expected answer shape—without exposing prompt templates or DSL specifics.

Retrieval + transformation → structured answers

  • Converts retrieved data into UI-ready JSON for tables/charts and a concise narrative summary.
  • Keeps output predictable for the frontend while allowing customized, context-aware variants.

Performance & cost controls

  • Template-backed responses and selective caching reduce LLM calls for recurring patterns.
  • Practical guardrails keep processing within budget/context limits; larger queries condense gracefully.

Insight generation

  • Adds short, plain-language takeaways aligned to the returned visualization.
  • Handles “no data” cases cleanly with helpful messaging.
